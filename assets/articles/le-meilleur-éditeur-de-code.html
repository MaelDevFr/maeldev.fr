Quel est LE meilleur éditeur de code ?

10m de lecture
Step 1 : Introduction aux éditeurs :
• Si vous êtes sur cette page ( très bon choix d'ailleurs ) c'est que vous cherchez l'éditeur de code parfait. J'en conclus que vous venez de terminer votre premier tuto youtube sur le code et que vous cherchez donc un bon éditeur de code pour bien commencer Et si vous êtes un peu plus curieux vous avez cherchez sur youtube quel est le meilleur éditeur de vous avez eu des réponses dans le style : "Le meilleur éditeur, c'est celui que vous avez" et vous en avez marre ; marre de ne pas avoir de réponses concrètes. Et bien sachez que vous êtes tombés au bon endroit ! Dans cette article je vais enfin révéler le nom de l'éditeur ultime, un secret parmis les développeurs ; secret bien gardé et durement accessible, je vais vous le révéler !
Step 2 : Mais avant, c'est quoi un éditeur de code ?
• Pour ceux qui savent déjà ce qu'est qu'un éditeur de code, voici un lien vers la suite mais pour ceux qui ne le saurait pas voici une mini introduction un éditeur de code est ni plus ni moint qu'un Word spécialisé pour le code. Il va prendre en compte la coloration syntaxique de votre code, le formatage, l'autocomplétion de ce que vous écrivez et bien plus encore Il est indispensable d'en avoir un ne serait-ce que pour le formatage du code qui le rend bien plus lisible.
Step 3 : Le meilleur éditeur de code est ...
• Et non, vous n'aurez pas la réponse tout de suite ! En effet, je vais procéder avec méthode pour trouver THE ONE éditeur de code. Déjà, formons une liste des éditeurs de code les plus populaires :
  • Visual Studio Code
  • IntelliJ
  • Sublim Text
  • Atom
  • Vim

Je traiterais donc de ces 5 éditeurs de code ( même si IntelliJ est un IDE )
Commençons justement par IntelliJ, l'IDE créer par JetBrains :
Points positifs :
  • C'est un IDE ( plus puissant qu'un éditeur de code )
  • Il est très puissant et propose des options que les autres ne propose pas
    ( noms de fonctions intelligents, débogage rapide, etc ... )
Points négatifs :
  • Sa version gratuite permet le développement uniquement en Java & KotLin
  • Sa version payante est certes puissantes mais chères pour un développeur venant de se lancer
  • Il est très lourd comparé aux autres ( 2,5 Go )
Bien, maintenant passons à Sublim Text créer par Sublim HQ
Points positifs :
  • Il est gratuit et supporte beaucoup de langages
  • Il est très léger et s'ouvre rapidement
  • Il propose une belle coloration syntaxique
Points négatifs :
  • Il est largement moins puissant que les autres
Passons désormais à Atom, un éditeur de code créer par Github
Points positifs :
  • Il est entièrement gratuit et supporte tous les langages connus
  • Il propose un marketplace gratuits d'extensions créer par la communauté majoritairement
  • Il propose une intégration de Git très poussée
Points négatifs :
  • Il prends plus de temps à s'ouvrir et peut parfois être un peu lent
Après cet éditeur de code récent, nous allons passé à Vim, un très vieil éditeur de code
Points positifs :
  • Il est complètement gratuit et supporte tous les langages connus ( parfois, il faudra utiliser un plugin )
  • Il passe par le terminal ce qui lui confère une extrême rapidité d'exécution
  • Il propose des plugins pour différents langages pour l'autocomplétion & le formatage
Points négatifs :
  • Il ne possède pas d'interfaces graphiques, tout est en ligne de commande ce qui le rend extrêmement dur à utiliser
Et maintenant, j'ai gardé le meilleur pour la fin, applaudissez : Visual Studio Code, créer par Microsoft
Points positifs :
  • Il est complètement gratuit et supporte tous les langages connus ( plus ou moins efficacement en fonction des extensions installées )
  • Possède un immense marketplace d'extensions gratuites créer majoritairement par la communauté
  • Il possède toutes les options ( ou presque ) d'un IDE, beaucoup de raccourcis clavier et un terminal intégré performant
  • Il est adapté pour tous types de projets ( grands ou petits )
Points négatifs :
  • Il est un peu lent au démarrage
  • Sans extensions vous ne pourrez pas profiter pleinement de ce qu'il peut vous offrir
Step 4 : Conclusion :
• Vous l'aurez compris, Visual Studio Code écrase la concurence... et de loin ! Il est LE meilleur dans sa catégorie et le restera ( tant qu'IntelliJ reste payant ). Si vous voulez tout de même téléchargez les éditeurs de code pour vous faire votre avis, voici les liens :

• IntelliJ
• Sublim Text
• Atom
• Vim
• Visual Studio Code
